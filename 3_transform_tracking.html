The Cdf Is the Bridge

Slide 21–22: X ~ Uniform(0,1), Y = −ln(1−X) ~ Exponential(1)

Background: transforming a random variable

Given X with a known cdf, and Y = g(X) for some function g, we can find Y’s cdf directly: FY(y) = P(Y ≤ y) = P(g(X) ≤ y), which is rewritten in terms of FX by solving the inequality for X. For the increasing transform g(x) = −ln(1−x) used here, this gives:

FY(y) = FX(x)  —  whenever y = g(x)

Because g is monotonic, {X ≤ x} and {Y ≤ y} are literally the same event — so their cdfs agree at that shared probability level. That identity is exactly the inverse-transform method for random number generation. Drag the slider below to watch it hold at every x.

Separately, the violet value y that comes out of the middle panel (its vertical axis) is the exact same number that goes into panel 3 (its horizontal axis) — panel 2’s output is panel 3’s input.

Because g is monotonic increasing, the events {X ≤ x} and {Y ≤ y} are literally the same event — so P(X ≤ x) = P(Y ≤ y), i.e. FX(x) = FY(y) for every value x. This identity is exactly the inverse-transform method: draw x as a realisation of Uniform(0,1); since FX(x) = x already is the target probability, y = g(x) lands precisely where FY reaches that same probability.
